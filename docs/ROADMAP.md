# GrapeNote 기획·로드맵

> 2026-07-20 사용자(제작자)와 합의한 제품 방향 문서. 세션이 바뀌어도 이 문서가 기준이다.
> 상태 표기: ✅ 완료 · 🔨 진행 중 · 📋 예정 · 💬 논의 필요

## 제품 정의

**"연습을 영상으로 검증하는 모든 그룹"을 위한 공개 서비스.**

- 온라인 악기 연습 진도 카드: 멤버가 연습 영상을 올리고, 검토자가 합격 판정하면 포도알이 채워진다.
- 타겟은 음악 학원 **과** 밴드·음악 동아리 둘 다 (범용). 개인 연습자는 "1인 그룹"으로 수용.
- 다른 그룹이 스스로 가입해서 쓰는 **공개 서비스**를 지향한다. (현재 실사용: 건국대 밴드 동아리 너드더락)

## 확정된 핵심 결정

| 주제 | 결정 |
|---|---|
| 용어 | **그룹 유형별 프리셋**. DB 스키마(`academies`, `teacher` role 등)는 그대로 두고 화면 표기만 교체. `academies.group_type`: `academy`(선생님/수강생) · `club`(운영진/멤버) · `other`(리더/멤버) |
| 저장 비용 | 당분간 Supabase 무료 티어 안에서 버틴다. **그룹마다 기본 용량 할당** → 프리미엄 결제 시 상향 |
| 수익 모델 | **그룹 단위 프리미엄**: 그룹장이 결제하면 그 그룹 전체에 프리미엄 기능 활성화 (용량↑, 화질↑, MR 재생 등). 결제 연동 전에는 `is_premium` 수동 토글로 실험 |
| 검토 병목 | 원인 = 밀린 영상 **개수**. 처방 = 음소거 **동시 재생 그리드** (영상 길이·손 움직임으로 연습 여부를 눈으로 훑고, 소리는 필요한 것만) |

## 로드맵

### Phase 1 — 검토 그리드 ✅ (2026-07-20 배포)

`/teacher/review`(선생님)와 `/me/review`(파트장) 검토함을 동시 재생 그리드로 개편.

- **1 / 2 / 4 / 6 분할 모드** 지원 (사용자 지정 스펙). 선택은 localStorage에 기억.
- 대기 영상이 타일로 깔리고 전부 **음소거 자동 재생**. 영상 길이 뱃지 표시.
- 소리는 한 번에 한 타일만: 타일의 🔊 버튼으로 전환.
- 타일 안에서 바로 🍇합격 / ↺재연습(코멘트 필수) 판정 → 판정된 타일 자리에 다음 대기 영상이 채워짐.
- 전역 배속(1x/1.5x/2x)이 모든 타일에 적용.
- 자세히 보고 싶으면 기존 상세 페이지(`…/review/[subId]`)로 이동하는 링크 유지.
- 판정 로직은 기존 `review_submission` RPC 재사용 — 마이그레이션 불필요.

### Phase 2 — 공개 서비스 기반 🔨 (2026-07-20 1차 구현)

- ✅ `academies.group_type`(0017) + `lib/terms.ts` 용어 프리셋 + `terms-server.ts`(getTerms, 요청당 캐시). 주요 화면 용어 교체 완료 — 학원=선생님/학생, 동아리=운영진/멤버, 그 외=리더/멤버. 조사(이/가) 문제가 있는 문구는 중립 표현으로 재작성. 너드더락 club 전환은 0017 적용 후.
- ✅ 온보딩: 그룹 유형 선택(학원/동아리·밴드/그 외) → 용어·플레이스홀더 즉시 반영.
- ✅ 랜딩 개편: 범용 카피 + 기능 3종 소개 + 너드더락 실사용 언급. 버튼 "멤버로 시작/그룹 만들기·리더 로그인".
- ✅ 저장 정책(lib/limits.ts): 그룹당 500MB — requestUpload에서 합계 검사, 대시보드 사용량 게이지(90%↑ 빨강). 판정 후 보존 30일→7일(cleanup cron).
- 📋 남은 것: 깊숙한 문구 일부(관리 패널 등)의 용어 교체 추가 다듬기.

### Phase 3 — 가시성과 동기부여 🔨 (2026-07-20 1차 구현, 사용자와 검토하며 다듬는 중)

- ✅ **완성·활동 피드**: 멤버 홈(/me)에 그룹 소식 — 포도송이 완성·포도알 합격 이벤트 (`get_group_feed` RPC, 0015). 이름·곡명·시각만 노출.
- ✅ **주간 연습왕 + 통계**: 피드 상단 👑 배너 + 리더용 /teacher/stats (멤버별 이번 주 제출/합격, 7일+ 무제출 💤 표시). `get_weekly_stats` RPC.
- ✅ **스트릭·배지**: 홈 🔥 연속 연습일 배너(오늘 안 올렸으면 "오늘 올리면 n+1일"), 포도밭에 배지 10종 (lib/badges.ts, 저장 없이 유도).
- ✅ 응원 리액션 (0018): feed_reactions 테이블, 피드 v2(target_id 포함)로 이벤트에 🔥👏🎉 토글. 남 명의 등록 RLS 차단.
- ✅ 멤버 간 현황 공개 (0018): academies.show_board + 대시보드 토글 → 멤버용 읽기 전용 /me/board (get_group_board RPC — 비공개면 빈 결과, 개수만 노출).
- 💬 사용자와 실사용 검토 후 다듬기로 함 ("나중에 나랑 검토하면서 고쳐나가게").

### Phase 4 — 알림 + 프리미엄 🔨

- 📋 **웹푸시 알림**: 새 제출 → 검토자, 판정/완성 → 멤버. (구독 테이블 + Web Push — 마지막 남은 큰 조각)
- ✅ **그룹 프리미엄 토글** (0018, 결제 없이 is_premium 수동): 무료 500MB/7일/50MB/480p ↔ 프리미엄 5GB/30일/200MB/720p (lib/limits.ts groupLimits). is_premium은 컬럼 권한(grant 목록 제외)으로 API 수정 차단 — 운영자(service role)만 변경. videos 버킷 상한 200MB 상향, cleanup cron 2단계 보존.
- 📋 프리미엄 잔여: **녹화 중 MR/반주 재생**, 영상 아카이브 다운로드, 고급 통계. 가격 월 4,900~9,900원 구간 추후 확정, 결제 수단 미정(토스페이먼츠 유력).
- ✅ **구글 드라이브 아카이브** (0019, 2026-07-20 구현): 그룹장이 대시보드에서 구글 연결(OAuth, drive.file 스코프) → 정리 크론이 삭제 직전에 'GrapeNote 아카이브' 폴더로 백업(`날짜_곡명_멤버_포도알N` 파일명). 백업 실패 시 삭제하지 않음(다음 실행 재시도, 유실 없음). refresh token은 정책 없는 drive_connections 테이블(service role 전용). 활성화 조건: GOOGLE_CLIENT_ID/SECRET 환경변수 (Google Cloud OAuth 클라이언트, 리디렉션 URI https://grapenote.vercel.app/api/google/callback).

### 저장 정책 확정 (2026-07-20, Supabase 무료 티어 1GB 대응)

- 사용량이 2.7GB(한도 2.7배)였음을 발견 → 사용자 결정: Pro 결제 없이 버티기.
- **판정 후 파일 보존 1일** (기록·코멘트 영구) + **앨범 업로드 차단, 인앱 촬영 전용**(480p 비트레이트 제한으로 파일 억제) + 그룹 한도 1GB.
- 수동 정리로 2.7GB → 1.1GB, 이후 하루치(~0.5GB)만 상시 유지되는 구조.

### 곡 중심 개편 🔨 (2026-07-20 1차 구현 — 팀 관리 UX 불편 해소)

사용자 피드백 "파트장 지정·곡 지정·팀원 지정이 불편" + "밴드는 악기별 세션장" + "곡 미션 명확화" + "곡별 MR"에 대한 응답. 마이그레이션 0016 (profiles.instrument, song_tracks 테이블+RLS, tracks 버킷).

- ✅ **곡 추가 마법사** (/teacher/songs/new): 곡명+미션+편성(악기별 그룹 체크)+포도알/기한 → 곡 팀 생성·멤버 소속·카드 배정 한 번에 (`createSong`, 멤버 먼저→카드 나중 트리거 순서 준수)
- ✅ **현황판 편성 수정**: 곡명 클릭 → 체크박스 모달 (`updateSongLineup` — 빼는 멤버 카드는 제출 기록 없을 때만 삭제, 기록 있으면 보존)
- ✅ **파트장 원탭**: 팀 관리에서 멤버 칩의 ☆/⭐ 탭으로 지정·해제
- ✅ **악기(세션)**: 학생 상세에서 악기 지정(프리셋+직접 입력) → 편성 화면 악기별 그룹핑, "악기 파트 팀 만들기" 버튼(`createInstrumentTeams`)으로 파트 팀 자동 생성 → ⭐로 세션장 지정
- ✅ **미션 강조**: 학생 카드 상단 "🎯 이번 곡 미션" 박스 (description 활용)
- ✅ **곡별 MR**: song_tracks — 선생님 업로드는 "⭐ 기본" 표시, 학생도 공유 업로드 가능. 학생 카드/선생님 카드 상세에서 재생·추가·삭제 (private 버킷 tracks, signed URL만)
- ✅ **곡 관리 화면** (/teacher/songs, 2026-07-20 추가): 곡별 편성·미션·기한·진행·MR 관리 허브. 설정 일괄 수정(updateSong), 곡 삭제(deleteSong — 곡 이름 입력 확인). 사용자 요청 "곡배정 버튼 말고도 곡 관리 화면도 있어야지"
- ✅ **멤버 홈 개편** (2026-07-20): 요약 카드 통합(포도알·포도송이·스트릭), 카드 우선순위 정렬(재연습→마감임박→검토중), 컴팩트 2열, 소식 접기. 사용자 피드백 "UI/UX 불편, 리더 화면은 관리에 용이해야" — 추가 다듬기 계속

## 하지 않기로 한 것

- 공연/발표 D-day — 이번 배치에서 제외 (추후 재논의 가능)
- DB 스키마의 용어 리네이밍 (`academies` → `groups` 등) — 화면 표기만 바꾼다
- Supabase Pro 선결제 — 사용자가 늘면 그때 결정

## 참고 (현재 시스템 요약)

- 스택: Next.js 16(App Router) + Supabase(Auth/Postgres RLS/Storage) + Vercel (https://grapenote.vercel.app)
- 제출 생성은 `create_submission` RPC, 판정은 `review_submission` RPC만 사용 (직접 insert/update 정책 없음)
- 팀 2층 구조: 너드/더락(파트장 검토용) + "🎵 곡명" 곡별 팀(편성 단위, 팀 숙제 자동 배정 트리거)
- 검증 문화: 기능 배치마다 `scripts/e2e-live.mjs` (실 Supabase 대상) 통과 후 배포
